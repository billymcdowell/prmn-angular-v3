# UI Design System — AI-Native Skill

This skill describes the **Angular UI design system** in `projects/ui`. Use it when building or refactoring UI: choose components, apply theming tokens, and compose layouts from the documented primitives.

---

## When to use this skill

- **Building UI:** You need buttons, forms, modals, tables, navigation, or feedback (toasts, loading).
- **Theming:** You need to change colors, radius, shadows, or typography across the app.
- **Choosing components:** You need to pick the right primitive (e.g. Select vs Autocomplete vs Combobox).
- **Composing layouts:** You need auth, dashboard, settings, marketing, or data-table patterns.

**Trigger phrases:** "use the design system", "use the UI library", "add a button/form/modal/table", "theme the app", "which component for X", "layout for login/dashboard/settings".

## Quick reference

- **Tokens:** [docs/THEMING-TOKENS.mdx](docs/THEMING-TOKENS.mdx) — colors, fonts, radius, shadows; override in `theme.css` or app.
- **Storybook:** Run the app’s Storybook to see live examples (paths match `Components/*` and `Examples/*`).
- **Imports:** Components are under `projects/ui/src/lib/<component>`; use the documented directives and elements from each component’s doc.
- **Forms:** Prefer [Form Field](docs/form-field.mdx) + [Input](docs/input.mdx) / [Textarea](docs/textarea.mdx) / [Select](docs/select.mdx) / [Checkbox](docs/checkbox.mdx) / [Switch](docs/switch.mdx) for consistent labels, hints, and errors.

When asked to implement or theme UI, read the relevant doc(s) above and use the listed APIs and tokens so that output stays consistent with this design system.
